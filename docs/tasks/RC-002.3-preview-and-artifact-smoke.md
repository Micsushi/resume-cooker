# RC-002.3: Preview And Artifact Separation Smoke

## Outcome

Local preview serves the latest valid public ATS PDF, reports failed rebuilds honestly, and never
confuses temporary preview with intentional saved output.

## Status Gate

**Blocked by RC-002.1 and RC-002.2.** Requires a browser or HTTP client and a usable build runtime.

## Owned Surfaces

- `generator/scripts/preview-server.mjs`
- `generator/public/index.html`
- preview tests
- `generator/README.md`
- `MACOS_COMPATIBILITY.md`

## Inputs And Outputs

Input: public ATS source and unused loopback port. Output: temporary PDF under `.runtime/preview`,
status/source/compile responses, and separately saved PDF only after explicit command.

## Scope

Exercise `/api/status`, `/api/source`, `/api/compile`, and `/preview.pdf`; test successful and failed
rebuilds, stale-artifact labeling, shutdown, and cleanup. Do not add editing UI or LAN binding.

## Sequence

1. Start preview on `127.0.0.1` with public source.
2. Verify initial status and PDF response.
3. Trigger one successful and one synthetic failed compile.
4. Confirm last-good versus current-failure state.
5. Stop server and inspect processes and artifact paths.

## Acceptance And Verification

- `npm run preview -- --source resume/source/ats.tex`
- HTTP endpoints return documented status and content types.
- Failed compile never claims current success or overwrites saved output.
- Server terminates cleanly; `git status --short` shows no generated tracked files.

## Failure And Privacy Constraints

Bind loopback only, sanitize compiler output, and never expose arbitrary filesystem content.

## Handoff Evidence

Endpoint matrix, successful/failed build IDs, artifact paths, shutdown evidence, and manual browser
inspection result.
