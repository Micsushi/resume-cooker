# RC-004.1: Comparison Fixtures And Normalized Facts

## Outcome

Public synthetic before/after fixtures and a deterministic normalization layer expose comparable
identity, education, experience, and skill facts with confidence metadata.

## Status Gate

**Ready after RC-006.1 publishes D4.** No PDF runtime required.

## Owned Surfaces

- `fixtures/compare/`
- `checker/scripts/compare-lib.mjs`
- `checker/scripts/compare-lib.test.mjs`

## Inputs And Outputs

Input: source LaTeX/text and optional schema-v1 structured JSON. Output: normalized fact records with
field type, entry index, source representation, confidence, and internal comparison value.

## Scope

Create synthetic fixtures for equivalent formatting and every planned regression. Normalize case,
whitespace, punctuation, LaTeX escapes, phone formatting, and dates. Structured JSON takes
precedence when supplied. Do not emit raw private values in findings or attempt external truth
verification.

## Sequence

1. Add fixture manifest and pass/fail cases.
2. Define normalized record and confidence shapes.
3. Implement structured and source extractors.
4. Test equivalent formatting, missing fields, ambiguous entries, and parse failure.

## Acceptance And Verification

- Fixtures contain no real person, employer, school, address, or contact value.
- Equivalent formatting normalizes identically.
- Source-only extraction carries lower confidence where structure is ambiguous.
- `node --test checker/scripts/compare-lib.test.mjs`
- fixture secret/private-value review documented.

## Failure And Privacy Constraints

Do not hash low-entropy private values as public evidence; hashes remain internal comparison tools
only. Missing input cannot produce “no regression.”

## Handoff Evidence

Fixture matrix, normalized schema example with synthetic data, confidence rules, and privacy review.
