# RC-008.3: Hunt Preflight Enforcement And Override

## Outcome

Hunt runs Resume Cooker preflight before Fletcher and applies D3 default policy with complete audited
override evidence.

## Status Gate

**Blocked by RC-008.2 and RC-006.1.** Local-only preflight does not require RC-005.

## Owned Surfaces

- Hunt C2 coordinator/policy/override/test paths from RC-008.1
- Hunt integration docs and synthetic fixtures

## Inputs And Outputs

Input: source path, optional PDF/JD/policy, adapter result, optional authorized override. Output:
proceed/block decision, original report reference, namespaced flags, and audit record.

## Scope

Invoke preflight before Fletcher. Default: pass proceeds, warnings proceed visibly, fail blocks;
unavailable/timeout/malformed block when required. Override records approver, reason, report identity,
and timestamp. Do not change Resume Cooker severity.

## Sequence

1. Add policy table tests for all statuses/errors.
2. Insert preflight before Fletcher invocation.
3. Implement persistent/reference-safe audit record.
4. Preserve original blocker evidence after override.
5. Add feature flag and local-only default.

## Acceptance And Verification

- Fletcher never runs before required preflight completes.
- Fail blocks without override; warnings remain visible.
- Override is explicit, attributable, and reversible.
- Hunt unit/integration tests for pass/warn/fail/unavailable/timeout/malformed/override.

## Failure And Privacy Constraints

Do not log raw reports, enable API mode from a key, or allow blanket/implicit overrides.

## Handoff Evidence

Policy matrix, coordinator call order, audit example with synthetic IDs, feature-gate behavior, and
privacy confirmation.
