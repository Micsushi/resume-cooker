# RC-006.3: Package Readiness And Release Gates

## Outcome

Work-package dependencies, stage exit gates, normal local profile, and strict release profile are
explicit and point only to existing packages or named external conditions.

## Status Gate

**Blocked by RC-006.1 and RC-006.2.**

## Owned Surfaces

- `docs/tasks/README.md`
- `docs/roadmap.md`
- private `current-handoff.md`

## Inputs And Outputs

Input: 46 package contracts and current verification. Output: package graph, stage gates, profile
requirements, and evidence-based live statuses.

## Scope

Replace whole-epic blocking where independent child work exists. Define normal versus strict
capability outcomes. Keep durable graph public and dated state private. Do not infer `done` from
commit messages.

## Sequence

1. Validate every package ID and direct dependency.
2. Build critical-path and grouped registry views.
3. Define stage entry/exit evidence.
4. Reconstruct current live statuses from commands/source.
5. Record exact next command.

## Acceptance And Verification

- Every dependency target exists or names an external condition.
- Blocked tasks state one exact unblock condition.
- Strict profile requires ATS-Checker and known page count; normal profile does not.
- Package-reference and local-link validation scripts return zero.

## Failure And Privacy Constraints

Do not place owners, credentials, generated artifacts, or transient machine failures in public task
contracts.

## Handoff Evidence

Graph validation, stage matrix, live status source evidence, current critical path, and next command.
