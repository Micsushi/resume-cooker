# RC-003.2: ATS-Checker Integration

## Outcome

ATS-Checker extracts a public sample PDF and produces normalized parser-agreement evidence against
baseline Poppler text.

## Status Gate

**Blocked by RC-003.1 and RC-002.2.** Requires supported Python environment and public ATS PDF.

## Owned Surfaces

- `checker/scripts/ats_checker_extract.py`
- `checker/scripts/ats-checker.mjs`
- `checker/scripts/ats-checker.test.mjs`
- `testers/ATS-Checker/README.md`
- `testers/README.md`

## Inputs And Outputs

Input: public PDF and baseline extracted text. Output: execution state, normalized token counts,
similarity, threshold, and sanitized disagreement finding.

## Scope

Document isolated dependency setup, execute existing helper, validate output, define token
normalization and threshold, and classify empty output, crash, absence, timeout, and disagreement.
Do not patch the vendored snapshot or expose extracted text.

## Sequence

1. Freeze supported Python version and environment setup.
2. Add command/path/error fixtures for Windows and POSIX.
3. Run helper against public ATS artifact.
4. Calibrate deterministic similarity threshold and document rationale.
5. Add adapter to strict profile.

## Acceptance And Verification

- Actual helper execution is distinguishable from skip.
- Similarity evidence contains counts and ratio, not raw tokens.
- Empty parser output never passes.
- `node --test checker/scripts/ats-checker.test.mjs`
- `npm run check:testers` with ATS-Checker environment active.

## Failure And Privacy Constraints

Never log PDF text, contact data, environment activation secrets, or full Python tracebacks in stable
reports.

## Handoff Evidence

Python/runtime versions, install command, actual invocation, similarity result, threshold, sanitized
failure sample, and artifact privacy statement.
