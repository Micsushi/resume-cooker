# RC-009.4: Cancellable Build And Preview State

## Outcome

Build operations have unique IDs, cancellation, stale-response protection, last-good artifact state,
and honest failed-current labeling.

## Status Gate

**Blocked by RC-001.2, RC-009.2, and RC-009.3.** Real build smoke requires RC-002.2.

## Owned Surfaces

- generator build operation service/handlers
- operation state and concurrency tests
- `docs/ui-contract.md`

## Inputs And Outputs

Input: source revision, requested engine/policy, operation ID/cancel signal. Output: running,
success, fail, cancelled state plus artifact revision and last-good/stale metadata.

## Scope

Serialize or safely supersede builds per source, associate artifacts with exact revisions, cancel
children, and keep last good PDF visibly stale after later failure. Do not save final output or alter
source automatically.

## Sequence

1. Add overlapping, out-of-order, fail-after-success, cancel, and timeout tests.
2. Implement operation registry and source-revision capture.
3. Connect cancellable build service.
4. Publish current/last-good artifact state.
5. Verify cleanup and artifact separation.

## Acceptance And Verification

- Older response cannot overwrite newer operation state.
- Failed current build never labels last-good PDF current.
- Cancel/timeout terminates child and reports typed state.
- Operation tests plus real public build smoke when runtime exists.

## Failure And Privacy Constraints

No orphan process, stale-current claim, implicit save, or raw compiler output in browser response.

## Handoff Evidence

State transition matrix, race tests, cancellation proof, artifact/revision mapping, and real-smoke
status.
