# RC-003.4: ats-screener Integration

## Outcome

The vendored ats-screener local scoring entrypoint evaluates public extracted text and JD fixtures,
returning validated component scores and missing-signal counts.

## Status Gate

**Blocked by RC-003.1 and RC-002.2.** RC-003.3 should finish first. Requires isolated install from
vendored lockfile.

## Owned Surfaces

- `checker/scripts/tester-wrapper.mjs`
- ats-screener adapter tests
- `testers/ats-screener/scripts/score-local.ts` only if integration defect requires upstream-local fix
- `testers/README.md`

## Inputs And Outputs

Input: public extracted resume text, public JD, local Node dependency environment. Output: validated
score components, missing-term count, execution state, and sanitized failure.

## Scope

Use only local scoring, define install/invocation, validate JSON schema, and classify missing `tsx`,
malformed JSON, nonzero exit, timeout, and score result. Do not invoke hosted APIs, Firebase, or UI.

## Sequence

1. Freeze lockfile-based install and exact local command.
2. Add output schema and runner fixtures.
3. Normalize scores without raw resume/JD terms.
4. Run public fixture smoke.
5. Document snapshot version and maintenance boundary.

## Acceptance And Verification

- Hosted/network paths are not invoked.
- Malformed or missing output cannot pass.
- Report identifies actual execution and score dimensions.
- `npm test -- --test-name-pattern="ats-screener"`
- `npm run check:testers` with isolated ats-screener dependencies installed.

## Failure And Privacy Constraints

Do not install tester packages at root, print source terms, or persist tester caches outside ignored
paths.

## Handoff Evidence

Install/invocation, schema version, component summary, network-free confirmation, skipped paths, and
cleanup instructions.
