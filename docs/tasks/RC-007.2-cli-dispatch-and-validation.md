# RC-007.2: Package Binary Dispatch And Validation

## Outcome

`resume-cooker` binary provides help/version, parses subcommands, and rejects invalid inputs before
creating outputs or launching tools.

## Status Gate

**Blocked by RC-007.1.**

## Owned Surfaces

- new CLI entrypoint under `cli/` or approved root script path
- `package.json` binary metadata
- CLI parser/dispatcher tests

## Inputs And Outputs

Input: argv and caller working directory. Output: validated command request or categorized usage
error; help/version on requested stream.

## Scope

Implement dependency-light parsing, help, version, unknown option/command handling, required path and
numeric validation, and caller-relative paths. Do not execute build/check/tester operations yet.

## Sequence

1. Add failing help/version/invalid fixtures.
2. Add `bin` metadata and dispatcher.
3. Validate paths/extensions/numbers before filesystem mutation.
4. Normalize usage errors to exit 64.
5. Test Windows and POSIX inputs.

## Acceptance And Verification

- Fresh root install can invoke help without global install.
- Invalid input creates no report/output directory.
- Unknown command and option exit 64.
- `npx resume-cooker --help`
- `npx resume-cooker --version`
- CLI parser tests and `npm run lint`.

## Failure And Privacy Constraints

Do not introduce heavy dependencies, resolve relative paths against module location, or echo private
arguments unnecessarily.

## Handoff Evidence

Help/version output, invalid matrix, package metadata, path tests, and created-file audit.
