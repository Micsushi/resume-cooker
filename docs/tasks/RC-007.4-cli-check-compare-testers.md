# RC-007.4: CLI Check, Compare, And Testers Commands

## Outcome

Stable CLI commands expose preflight suites, postflight comparison, and tester profiles without
duplicating core rules.

## Status Gate

**Blocked by RC-003.6, RC-004.6, and RC-007.2.** API mode additionally depends on existing explicit
consent configuration; live readiness is not required.

## Owned Surfaces

- CLI command handlers
- checker module exports needed by handlers
- command-level tests and public fixtures
- `package.json` compatibility aliases

## Inputs And Outputs

Input: validated source/PDF/JD/structured/profile options. Output: schema-v1 reports and D7 exit code.

## Scope

Map `check --suite local|api|full`, `compare`, and `testers --profile normal|strict`. Preserve API
opt-in, postflight precedence, skip semantics, and content-left-machine metadata. Do not implement
Hunt policy.

## Sequence

1. Add passing/warning/failing public command fixtures.
2. Implement check dispatcher and consent validation.
3. Implement compare dispatcher with optional structured/PDF/JD inputs.
4. Implement tester profile dispatcher.
5. Keep npm aliases working or document exact migration.

## Acceptance And Verification

- API key without explicit consent sends no request.
- Compare fact regression exits 2; warnings exit 0.
- Strict missing ATS-Checker exits 69.
- CLI command tests, public local check, public compare pass/fail, normal tester run.

## Failure And Privacy Constraints

Never import Hunt, expose raw content, or convert optional skip into executed pass.

## Handoff Evidence

Per-command matrix, schema examples, exits, privacy metadata, compatibility aliases, and skipped
external paths.
