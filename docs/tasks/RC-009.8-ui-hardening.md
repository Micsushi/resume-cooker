# RC-009.8: UI Security, Accessibility, And Lifecycle Hardening

## Outcome

Complete local UI passes path, request, rendering, concurrency, shutdown, accessibility, and browser
end-to-end gates on public fixtures.

## Status Gate

**Blocked by RC-009.7.** Manual macOS build smoke requires RC-002.3.

## Owned Surfaces

- all local UI server/frontend tests and security middleware
- browser automation configuration
- `ui-design.md`, `docs/ui-contract.md`, compatibility docs
- private handoff for host evidence

## Inputs And Outputs

Input: complete v1 UI and public malicious/normal fixtures. Output: security, accessibility,
concurrency, performance-readiness, lifecycle, and platform evidence.

## Scope

Test loopback binding, traversal/symlink escape, CSRF, HTML injection, unsupported files, stale
operations, cancellation, output overwrite, keyboard/screen-reader labels, server shutdown, and child
cleanup. Native packaging remains deferred.

## Sequence

1. Add adversarial request/file/render fixtures.
2. Run backend security and race tests.
3. Run browser accessibility and keyboard suite.
4. Run full public end-to-end workflow and shutdown checks.
5. Run manual macOS smoke; document untested platforms honestly.

## Acceptance And Verification

- Server binds `127.0.0.1` by default.
- All unsafe path/request/render cases reject safely.
- No stale operation wins; no child remains after shutdown.
- Keyboard workflow covers edit/save/build/check/findings/output.
- Root tests, browser suite, accessibility audit, manual smoke, and `git status` pass.

## Failure And Privacy Constraints

No private fixture, API key, generated output, screenshot with personal data, or LAN exposure.

## Handoff Evidence

Security matrix, accessibility results, browser artifacts using synthetic data, process cleanup,
platform table, ignored artifacts, and deferred native-packaging statement.
