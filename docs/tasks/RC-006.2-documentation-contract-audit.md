# RC-006.2: Documentation Contract Audit

## Outcome

All public project documents describe accepted D1-D8 behavior consistently and label capabilities
implemented, partial, planned, or deferred.

## Status Gate

**Blocked by RC-006.1.** No product-code changes required.

## Owned Surfaces

- `README.md`, `ui-design.md`, compatibility documents
- `docs/*.md`, `docs/tasks/*.md`
- `generator/README.md`, `testers/README.md`

## Inputs And Outputs

Input: accepted decisions and current code evidence. Output: aligned docs plus reviewed contradiction
search.

## Scope

Audit ATS ownership, page severity, pre/post enforcement, fact precedence, tester requiredness, API
scope, CLI compatibility, and UI ownership. Preserve accurate operational details. Do not mark
mock-only or smoke-only behavior complete.

## Sequence

1. Build decision-to-document audit matrix.
2. Patch contradictory or unlabeled claims.
3. Link detailed behavior to decision and package pages.
4. Search policy phrases and manually classify every result.
5. Format changed docs.

## Acceptance And Verification

- No document calls API or third-party testers default root behavior.
- Page-limit language matches D2.
- UI and Hunt boundaries match D3/D8.
- `rg -n -i "warn only|hard fail|choose one|open question|API.*default" README.md docs generator/README.md MACOS_COMPATIBILITY.md LINUX_COMPATIBILITY.md ui-design.md`
- `npx prettier --check` on changed docs.

## Failure And Privacy Constraints

Do not erase useful future intent; label it. Do not add live assignments, host state, or private
values to public docs.

## Handoff Evidence

Audit matrix, changed documents, explained search hits, unresolved external evidence, and formatting
result.
