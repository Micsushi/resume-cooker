# RC-005.2: OpenRouter Live Validation

## Outcome

One authorized OpenRouter model completes a bounded synthetic review and all local guardrail/error
paths produce correct sanitized reports.

## Status Gate

**Blocked by RC-005.1** plus a scoped `OPENROUTER_API_KEY` and network access.

## Owned Surfaces

- `checker/scripts/api-providers.mjs`
- `checker/scripts/api-analysis.mjs`
- related provider tests and synthetic response shapes
- private project handoff

## Inputs And Outputs

Input: authorized model, key via environment, public fixtures, exact limits. Output: sanitized report
status, provider/model metadata, request count, approximate cost, and content-left-machine state.

## Scope

Run disabled/key-present, enabled/key-missing, one success, oversized local rejection, approved
provider error, and optional short-timeout cases within budget. Fix only contract defects observed.
Do not benchmark models or retain raw responses.

## Sequence

1. Re-run network-free provider tests.
2. Execute zero-request guardrail cases.
3. Execute one authorized successful review.
4. Exercise additional error case only if budget permits.
5. Audit report, console, diff, and history for secrets/content.

## Acceptance And Verification

- Current model identifier authenticates and returns parseable findings.
- `content_left_machine` becomes true only after attempted request.
- Provider/timeout errors become warnings, not suite crashes.
- Request count stays within authorization.
- `npm run check:api` under approved environment.

## Failure And Privacy Constraints

No private source, key, raw prompt/response, provider error body, or retry beyond ceiling is retained.

## Handoff Evidence

Model, request count, approximate cost, sanitized check counts, parsing result, guardrail cases, and
secret audit.
