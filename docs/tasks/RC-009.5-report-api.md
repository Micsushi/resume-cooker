# RC-009.5: Local Report Execution API

## Outcome

Local UI server runs versioned Resume Cooker checks and returns sanitized report summaries with
explicit pass, warning, fail, skip, capability, and privacy states.

## Status Gate

**Blocked by RC-007.4, RC-007.5, and RC-009.2.**

## Owned Surfaces

- generator/local UI report service and handlers
- report endpoint tests
- `docs/ui-contract.md`

## Inputs And Outputs

Input: approved source/artifact/JD identifiers, suite/profile, operation ID, optional explicit API
authorization. Output: running/completed/cancelled state and sanitized schema-v1 report projection.

## Scope

Invoke stable backend boundary, validate schema/version, group findings by severity/category, preserve
skip and content-left-machine state, and support cancellation. API suite requires separate explicit
confirmation. Do not duplicate scoring in UI server.

## Sequence

1. Add pass/warning/fail/skip/malformed/cancel fixtures.
2. Implement report operation service and schema validation.
3. Add approved input resolution and local suite.
4. Add explicit API-confirmation token flow without key exposure.
5. Sanitize detailed errors and cleanup.

## Acceptance And Verification

- Skips are distinct from pass.
- Malformed/unsupported report cannot appear successful.
- No API request occurs from key presence alone.
- Report service/endpoint tests and local public-fixture smoke.

## Failure And Privacy Constraints

Never send keys to browser, render raw tool output, persist raw content beyond approved local policy,
or allow arbitrary paths.

## Handoff Evidence

Endpoint/state matrix, schema compatibility, cancellation, API-declined proof, privacy projection, and
smoke result.
