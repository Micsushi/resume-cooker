# RC-002.1: Reproducible macOS Runtime

## Outcome

One supported macOS PDF runtime is installed, identified, and reproducible by a second contributor.

## Status Gate

**Blocked by RC-001.4** and access to a macOS host. Choose native TeX plus Poppler or Docker Desktop;
proving both is optional.

## Owned Surfaces

- `MACOS_COMPATIBILITY.md`
- private project handoff

## Inputs And Outputs

Input: macOS version, architecture, selected runtime, exact tool/image versions. Output: verified setup
instructions and capability evidence.

## Scope

Select runtime, record why, verify CLI visibility from terminal and GUI-launched processes, and
confirm required capabilities. Do not build private resumes, install every tester, or pin images
without a separate reproducibility decision.

## Sequence

1. Record host baseline and selected runtime.
2. Follow documented install/setup from a clean shell.
3. Run capability probes and strict engine requirement.
4. Verify filesystem access for repository mounts when using Docker.
5. Update exact recovery steps from observed failures.

## Acceptance And Verification

- `npm run check:tools` reports selected engine and PDF text tools usable.
- `npm run check:tools -- --require-pdf-engine` exits zero.
- Apple Silicon compatibility is recorded when applicable.
- A second shell reproduces capability results.

## Failure And Privacy Constraints

No personal source, generated PDF, Homebrew token, Docker credential, or absolute home path enters
public docs.

## Handoff Evidence

Runtime choice, versions, command results, setup deviations, untested alternative, and next command
for RC-002.2.
