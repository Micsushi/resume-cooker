# RC-004.4: JD Addition Grounding

## Outcome

Material additions in a tailored resume are classified as grounded, ambiguous, or ungrounded using
source/profile evidence; JD presence alone never proves a fact.

## Status Gate

**Blocked by RC-004.1.** Structured profile evidence is optional but improves confidence.

## Owned Surfaces

- `checker/scripts/compare-lib.mjs`
- `checker/scripts/compare-lib.test.mjs`
- `checker/scripts/jd-analysis.mjs`
- `fixtures/compare/`

## Inputs And Outputs

Input: source/tailored facts, optional approved profile facts, optional JD signal set. Output: stable
grounded, ambiguous-synonym, and ungrounded-addition findings.

## Scope

Identify material tailored-only facts, match against source/profile evidence, and use JD only to
prioritize relevance. Maintain conservative synonym mapping with review confidence. Do not use model
review in deterministic comparison.

## Sequence

1. Add grounded, ungrounded, JD-only, synonym, and nonmaterial fixtures.
2. Define materiality and source-evidence rules.
3. Implement deterministic exact and approved-synonym matching.
4. Emit review warnings for ambiguous mappings and blockers for clearly fabricated configured facts.

## Acceptance And Verification

- JD-only skill addition is not grounded.
- Approved profile fact grounds an addition even if absent from source resume.
- Ambiguous synonym never silently passes.
- `node --test checker/scripts/compare-lib.test.mjs checker/scripts/jd-analysis.test.mjs`
- Run grounded and ungrounded compare fixtures.

## Failure And Privacy Constraints

Do not store raw JD/resume excerpts, expand synonyms dynamically, or claim external truth.

## Handoff Evidence

Materiality rules, evidence precedence, synonym list ownership, fixture results, and sanitized
findings.
