# RC-003.6: Tester Profiles And Cross-Parser Summary

## Outcome

Normal and strict tester profiles produce predictable required/optional behavior and one sanitized
cross-parser summary.

## Status Gate

**Blocked by RC-003.2 and RC-003.3.** RC-003.4 and RC-003.5 may remain optional or deferred.

## Owned Surfaces

- `checker/scripts/tester-runner.mjs`
- tester runner tests
- `package.json`
- `docs/evaluation-suites.md`
- `testers/README.md`

## Inputs And Outputs

Input: selected profile plus public PDF/text/JD. Output: per-adapter execution states, requiredness,
cross-parser agreement, overall report status, and missing-capability category.

## Scope

Normal profile treats all tester adapters as optional. Strict release profile requires ATS-Checker
and a known page count; other adapters remain optional until a new decision. Aggregate execution
without averaging incompatible scores.

## Sequence

1. Add profile configuration and requiredness fixtures.
2. Aggregate adapter states separately from findings.
3. Implement strict unavailable-capability result.
4. Add cross-parser structural summary.
5. Document commands and dependency setup.

## Acceptance And Verification

- Skip never appears as executed pass.
- Normal profile completes with explicit warnings when dependencies are absent.
- Strict profile returns unavailable capability when ATS-Checker cannot run.
- `node --test checker/scripts/*tester*.test.mjs`
- `npm run check:testers`
- strict-profile command documented by this package.

## Failure And Privacy Constraints

Do not combine unrelated scores into one authoritative number or include raw parsed content.

## Handoff Evidence

Profile matrices with dependencies present/absent, overall statuses/exits, cross-parser summary, and
documentation links.
