# RC-009.3: Safe Source Load And Save

## Outcome

Local API loads and explicitly saves approved raw-LaTeX sources with traversal protection and
external-modification conflict detection.

## Status Gate

**Blocked by RC-009.2.**

## Owned Surfaces

- generator source-file service and handlers
- path/security/concurrency tests
- `docs/ui-contract.md`

## Inputs And Outputs

Input: configured roots, relative source ID, optional new text, expected revision token. Output:
source text/metadata or typed not-found, unsupported, forbidden, conflict, and write error.

## Scope

Resolve real paths inside approved roots, allow approved `.tex` files, use revision token for saves,
and perform safe replacement. Reject traversal, symlink escape, unsupported files, and stale writes.
Do not offer arbitrary filesystem picker or autosave.

## Sequence

1. Add load/save/traversal/symlink/conflict fixtures.
2. Implement canonical approved-root resolution.
3. Implement load metadata and revision token.
4. Implement explicit conflict-aware save.
5. Sanitize path errors and document recovery.

## Acceptance And Verification

- Traversal and symlink escape are rejected.
- External changes return conflict without overwrite.
- Successful save updates revision and dirty state contract.
- Source service/security tests and `npm run lint`.

## Failure And Privacy Constraints

Never return arbitrary paths, auto-create outside approved roots, follow escaping symlinks, or log
source text.

## Handoff Evidence

Path policy, test matrix, conflict example, write strategy, supported roots/extensions, and recovery.
