# RC-002.2: ATS PDF Build And Text Pipeline

## Outcome

The public ATS source builds into a non-empty PDF whose page count, extracted text, ordered sections,
encoding, and parser agreement are classified in one local report.

## Status Gate

**Blocked by RC-002.1.** Parser agreement remains an explicit optional skip until RC-003.2; this
package supplies the public PDF needed by that later integration.

## Owned Surfaces

- `resume/source/ats.tex` only for build defects, never claim edits
- `checker/scripts/check.mjs`
- `checker/scripts/text-layer.mjs`
- public synthetic fixtures and related tests
- `MACOS_COMPATIBILITY.md`

## Inputs And Outputs

Input: public ATS source and selected runtime. Ignored outputs: `resume/output/ats.pdf`,
`resume/output/ats.txt`, `.runtime/reports/local-ats-check.json`.

## Scope

Build, count, extract, validate ordered sections and encoding, apply D2 one-page policy, and run
ATS-Checker agreement when available. Do not rewrite claims or upload artifacts.

## Sequence

1. Build ATS PDF and verify non-empty artifact.
2. Extract text and count pages.
3. Run deterministic checks and inspect every finding.
4. Run parser agreement or record strict-profile unavailable capability.
5. Manually inspect selectable text and reading order.

## Acceptance And Verification

- `npm run build:pdf:ats`
- `npm run check:local:ats`
- PDF page count is known and within configured policy.
- Extracted standard sections are present and ordered; no replacement-character noise exists.
- Report status and command exit agree with D2 and D7.

## Failure And Privacy Constraints

Never weaken checks to fit a failing artifact, print contact values, or stage generated files.

## Handoff Evidence

Tool used, page count, sanitized check counts, parser similarity metadata, manual inspection results,
ignored artifacts, and `git status`.
