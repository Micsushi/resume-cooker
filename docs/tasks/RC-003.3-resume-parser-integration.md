# RC-003.3: ResumeParser Integration

## Outcome

ResumeParser runs locally against the public ATS PDF and reports sanitized structural-presence
evidence for contact, education, experience, and skills.

## Status Gate

**Blocked by RC-003.1 and RC-002.2.** RC-003.2 should finish first to stabilize parser evidence
patterns.

## Owned Surfaces

- `checker/scripts/tester-wrapper.mjs`
- ResumeParser adapter tests
- `testers/ResumeParser/README.md`
- `testers/README.md`

## Inputs And Outputs

Input: public PDF and isolated Python runtime. Output: section-presence booleans, entry counts, adapter
state, duration, and sanitized errors.

## Scope

Define supported Python version, isolated install, exact profile command, output parser, and mapping
for missing models/data, import errors, malformed output, timeout, and parsed structure. Do not retain
field values or modify vendored parser logic.

## Sequence

1. Record supported runtime and dependency command.
2. Add Windows/POSIX environment and command tests.
3. Validate structured output schema.
4. Map presence/count evidence into normalized findings.
5. Run public artifact smoke.

## Acceptance And Verification

- Contact parseability is reported without values.
- Missing assets differ from parser disagreement.
- Import failure and timeout are explicit skips or failures according to requestedness.
- `npm test -- --test-name-pattern="ResumeParser"`
- `npm run check:testers` with ResumeParser environment active.

## Failure And Privacy Constraints

No names, addresses, employers, education values, or raw parser objects enter console/report evidence.

## Handoff Evidence

Runtime/setup, command, section/count summary, sanitized error examples, supported platforms, and
unexercised paths.
