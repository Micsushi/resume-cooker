# RC-009.2: Preview Operation And Request Services

## Outcome

Existing preview behavior is separated into testable operation services and HTTP handlers without
changing public behavior.

## Status Gate

**Blocked by RC-002.3 and RC-009.1.**

## Owned Surfaces

- `generator/scripts/preview-server.mjs`
- new generator service/handler modules
- preview unit/integration tests

## Inputs And Outputs

Input: approved server config, source path, build service, request. Output: typed handler response and
operation state; server startup/shutdown lifecycle.

## Scope

Extract routing, source/build/status/artifact services, dependency injection, sanitized errors, and
shutdown cleanup. Preserve endpoints and loopback default. Do not add editing or new frontend.

## Sequence

1. Add characterization tests for current endpoints.
2. Extract build/status service with injected runner/clock/IDs.
3. Extract request handlers and response normalization.
4. Centralize startup, loopback binding, and shutdown.
5. Re-run RC-002.3 smoke.

## Acceptance And Verification

- Existing preview endpoint behavior remains compatible.
- Handlers test without real TeX/server socket.
- Shutdown terminates child builds.
- Preview unit tests, `npm run lint`, and real preview smoke when runtime exists.

## Failure And Privacy Constraints

Do not expose compiler/source details in public errors, change save semantics, or introduce framework
dependencies without need.

## Handoff Evidence

Characterization matrix, extracted boundaries, lifecycle tests, smoke result, and compatibility notes.
