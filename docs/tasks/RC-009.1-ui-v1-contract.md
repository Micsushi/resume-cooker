# RC-009.1: UI v1 Source, Save, Security, And State Contract

## Outcome

UI v1 has frozen raw-LaTeX ownership, approved-root file rules, operation state model, endpoint
contract, and public browser fixtures before implementation.

## Status Gate

**Blocked by RC-006.1 and RC-007.1.** Contract design may use existing preview behavior without a
proven macOS runtime.

## Owned Surfaces

- `ui-design.md`
- `docs/ui-contract.md`
- public UI fixture sources and expected states
- endpoint/schema contract tests

## Inputs And Outputs

Input: D1, D6, D7, D8 and current preview endpoints. Output: approved endpoint, state, path, save,
conflict, build, report, privacy, and output-save contracts.

## Scope

UI edits raw LaTeX within configured roots. Define dirty/saved/conflict state; operation IDs;
build/check cancellation; stale results; temporary/saved artifacts; loopback/CSRF/path controls; API
confirmation. Defer structured editing, rewriting, cloud, and native packaging.

## Sequence

1. Freeze user flows and state transitions.
2. Define endpoint inputs/errors and approved-root policy.
3. Define optimistic concurrency token and operation identity.
4. Define report/tool/privacy display contracts.
5. Add fixture/state contract tests.

## Acceptance And Verification

- Every mutating action is explicit and CSRF-protected.
- Failed/current/last-good states are distinguishable.
- External file change cannot be overwritten silently.
- Contract fixture tests and `npx prettier --check ui-design.md docs/ui-contract.md`.

## Failure And Privacy Constraints

No arbitrary file access, LAN binding, browser-stored key, implicit API call, or autosave overwrite.

## Handoff Evidence

State table, endpoint table, path/save policy, security decisions, fixtures, and deferred features.
