# RC-009.7: Findings, Tool Readiness, Privacy, And Saved Output UI

## Outcome

UI displays categorized checks, skipped evidence, tool recovery, content-left-machine state, and an
explicit saved-output workflow without duplicating backend policy.

## Status Gate

**Blocked by RC-009.5 and RC-009.6.** API authorization display also depends on D6; live provider
readiness is not required.

## Owned Surfaces

- local UI findings/tool/output components
- output-save handler/service using stable build boundary
- component/browser tests
- `ui-design.md`, `docs/ui-contract.md`

## Inputs And Outputs

Input: report projection, capability state, current source/build revision, explicit output request.
Output: findings navigation, recovery guidance, privacy badge, confirmation, and verified saved PDF.

## Scope

Group by category/severity, distinguish skip, link to safe context, display actionable capability
reasons, show local/external status, and require visible output path/overwrite confirmation. API review
uses explicit confirmation. Do not recalculate statuses in frontend.

## Sequence

1. Add report/capability display fixtures.
2. Build findings navigation and skip treatment.
3. Build tool recovery and privacy state.
4. Add saved-output/overwrite confirmation and artifact verification.
5. Add API declined/authorized synthetic flows.

## Acceptance And Verification

- Skip never appears green/pass.
- Backend status remains source of truth.
- Saved output requires explicit action and verified non-empty artifact.
- API declined sends nothing.
- Component/browser tests plus public local-check smoke.

## Failure And Privacy Constraints

Never expose key, raw report content, unsafe HTML, silent overwrite, or false saved-artifact success.

## Handoff Evidence

Synthetic screenshots, findings/skip matrix, recovery states, privacy flows, output verification, and
browser tests.
