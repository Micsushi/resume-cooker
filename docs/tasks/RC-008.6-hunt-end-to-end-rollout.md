# RC-008.6: Hunt End-To-End Rollout And Rollback

## Outcome

Public fixtures prove enabled, overridden, unavailable, failed, and disabled integration paths, with
a documented rollout and rollback that preserves Fletcher's prior behavior.

## Status Gate

**Blocked by RC-008.5.** API-backed path additionally requires provider readiness from RC-005.4.

## Owned Surfaces

- Hunt end-to-end tests and feature configuration
- Resume Cooker/Hunt integration docs
- private handoffs for both projects

## Inputs And Outputs

Input: packaged CLI, Hunt feature gate, public source/JD/Fletcher fixtures. Output: end-to-end evidence,
rollout stages, monitoring signals, rollback command/config, and remaining constraints.

## Scope

Run local-only success, warning, source defect, tailoring regression, unavailable executable,
timeout, override, and feature-disabled flows. API path is separate. Confirm rollback disables calls
without breaking Fletcher. Do not enable by default without explicit release authorization.

## Sequence

1. Add complete public end-to-end matrix.
2. Run local-only enabled cases.
3. Run unavailable/timeout/override cases.
4. Run feature-disabled baseline and compare behavior.
5. Document staged enablement, metrics, recovery, and rollback.

## Acceptance And Verification

- Source defects and Fletcher regressions remain distinguishable.
- Disabled mode matches prior Fletcher path.
- No missing/malformed report becomes ready.
- Both repos' unit/integration suites and one local end-to-end run pass.

## Failure And Privacy Constraints

No private artifacts enter CI or hosted logs. Do not couple rollout to API validation.

## Handoff Evidence

Matrix results, feature settings, rollback proof, report/flag versions, privacy status, and explicit
release decision still required.
