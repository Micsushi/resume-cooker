# RC-005.3: Anthropic Live Validation

## Outcome

One authorized Anthropic model completes a bounded synthetic review and validates adapter behavior
against the live response envelope.

## Status Gate

**Blocked by RC-005.1** plus a scoped `ANTHROPIC_API_KEY` and network access. Independent from
RC-005.2 after guardrails are frozen.

## Owned Surfaces

- `checker/scripts/api-providers.mjs`
- `checker/scripts/api-analysis.mjs`
- related provider tests and synthetic response shapes
- private project handoff

## Inputs And Outputs

Input: authorized Anthropic model, key via environment, public fixtures, limits. Output: sanitized
provider/model/request metadata, normalized findings, approximate cost, and privacy state.

## Scope

Repeat the authorized success and zero-request guardrail matrix against Anthropic. Validate actual
content blocks and parse behavior. Exercise timeout/provider error only when explicitly included in
budget. Do not compare model quality or make Anthropic default.

## Sequence

1. Run network-free Anthropic tests.
2. Confirm disabled/key-present sends nothing.
3. Run one authorized success.
4. Fix and retest response-contract defects within remaining budget.
5. Perform secret/content audit and remove transient output.

## Acceptance And Verification

- Authorized model returns normalized findings.
- Disabled and local-rejection paths retain `content_left_machine: false`.
- Attempted request records true even when provider fails.
- Request count stays within authorization.
- `RESUME_COOKER_API_PROVIDER=anthropic npm run check:api` with approved environment.

## Failure And Privacy Constraints

Same guardrails as RC-005.2; never prescribe or persist literal credentials.

## Handoff Evidence

Model, count, cost, sanitized result, response-envelope compatibility, error paths, and secret audit.
