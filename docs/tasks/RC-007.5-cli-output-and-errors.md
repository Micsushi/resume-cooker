# RC-007.5: CLI Output, Errors, Reports, And Interruptions

## Outcome

All CLI commands obey JSON-only stdout, sanitized stderr, report-file equivalence, categorized errors,
and reliable interruption cleanup.

## Status Gate

**Blocked by RC-007.3 and RC-007.4.**

## Owned Surfaces

- CLI output/error/lifecycle utilities
- command-level integration tests
- `docs/cli-contract.md`

## Inputs And Outputs

Input: command result/error/signal. Output: one machine or human response, optional report file, stable
exit code, and cleaned resources.

## Scope

Centralize stream routing and errors. Validate report written to file matches stdout run identity and
status. Sanitize tool/provider errors. Handle SIGINT/SIGTERM for preview and child testers.

## Sequence

1. Add stdout contamination and sanitizer tests.
2. Implement result serialization and stderr routing.
3. Normalize usage, quality, capability, and internal errors.
4. Add atomic report writing and equivalence checks.
5. Add interruption cleanup tests.

## Acceptance And Verification

- Every JSON-mode stdout parses with no prefix/suffix.
- Report file and stdout agree on schema/status/run ID.
- Signals terminate children and return documented exit behavior.
- CLI output/error integration tests and `git diff --check`.

## Failure And Privacy Constraints

No raw provider body, compiler source excerpt, secret, or machine-local absolute path enters stable
output.

## Handoff Evidence

Stream captures, error/exit matrix, sanitizer fixtures, atomic-write behavior, and process cleanup
results.
