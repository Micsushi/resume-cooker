# RC-008.1: Hunt And Fletcher Surface Inventory

## Outcome

Current Hunt/Fletcher/C3 orchestration files, data contracts, test harnesses, feature gates, and
artifact storage boundaries are mapped before integration code is designed.

## Status Gate

**Blocked until the current Hunt repository is available.** Resume Cooker CLI fixture work may
continue independently.

## Owned Surfaces

- `docs/hunt-c2-integration-notes.md`
- synthetic integration fixtures in Resume Cooker
- Hunt-side design note or issue identified during inventory

## Inputs And Outputs

Input: current Hunt repository/source/tests and RC schema-v1 fixtures. Output: exact file/function
inventory, data-flow diagram, constraint list, and package dependency updates.

## Scope

Locate C2 entry, Fletcher inputs/outputs/flags, C3 readiness, override/audit facilities, subprocess
utilities, cancellation, storage, and tests. Record paths from evidence. Do not modify Hunt or invent
an architecture.

## Sequence

1. Read Hunt router/docs and repository status.
2. Trace one public source through C2/Fletcher/C3.
3. Identify insertion points and current failure/override behavior.
4. Freeze synthetic report fixtures and compatibility assumptions.
5. Update integration notes with exact surfaces.

## Acceptance And Verification

- Every proposed integration point links to an existing path/symbol/test.
- Existing Fletcher flags and C3 readiness semantics are documented.
- Unknowns become named blockers, not assumptions.
- Read-only Hunt tests/trace commands recorded.

## Failure And Privacy Constraints

Do not copy private Hunt artifacts or local absolute paths into public fixtures. Do not alter either
repository in this inventory package.

## Handoff Evidence

Repository revision, path/symbol map, current flow, fixture list, blockers, and first failing Hunt
test for RC-008.2.
