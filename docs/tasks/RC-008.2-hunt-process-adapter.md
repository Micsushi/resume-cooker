# RC-008.2: Hunt Process Adapter

## Outcome

Hunt invokes Resume Cooker through its packaged CLI with timeout, cancellation, schema validation,
sanitized errors, and no internal imports.

## Status Gate

**Blocked by RC-007.6 and RC-008.1.**

## Owned Surfaces

- exact Hunt adapter/config/test paths recorded by RC-008.1
- Resume Cooker synthetic schema-v1 fixtures
- Hunt integration documentation

## Inputs And Outputs

Input: executable/config, validated paths, command options, timeout/cancellation context. Output:
validated schema-v1 report or typed unavailable, timeout, cancellation, malformed, or unsupported-
schema result.

## Scope

Build one subprocess boundary, capture bounded sanitized streams, validate schema/version and
exit/report agreement, and clean children. Do not apply preflight/postflight policy in adapter.

## Sequence

1. Add fixture-driven adapter tests for every terminal outcome.
2. Implement command construction and path handling.
3. Add timeout/cancellation and child cleanup.
4. Validate schema, status, exit, and content-left-machine.
5. Sanitize persisted/logged errors.

## Acceptance And Verification

- No Resume Cooker module import exists in Hunt.
- Missing executable, timeout, invalid JSON, unsupported version, and exit mismatch are typed failures.
- Cancellation leaves no child.
- Hunt adapter unit tests and Resume Cooker CLI fixture tests pass.

## Failure And Privacy Constraints

Never persist raw stderr, embed source/JD content in argv when paths suffice, or treat malformed
output as pass.

## Handoff Evidence

Exact Hunt files, command fixture matrix, timeout/cancel results, sanitizer behavior, and supported
schema versions.
