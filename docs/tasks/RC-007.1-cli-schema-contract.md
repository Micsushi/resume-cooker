# RC-007.1: CLI Grammar And Schema-v1 Contract

## Outcome

Stable command grammar, report fixtures, stdout/stderr behavior, and exit codes exist before binary
implementation.

## Status Gate

**Blocked by RC-004.6 and RC-006.1.** Contract fixture work for existing check reports may begin
independently.

## Owned Surfaces

- `docs/cli-contract.md`
- `fixtures/cli/`
- CLI contract tests
- report schema/fixture files

## Inputs And Outputs

Input: existing commands and D7. Output: grammar for `tools`, `build`, `preview`, `check`, `compare`,
and `testers`; schema-v1 fixtures; exit matrix `0/2/64/69/70`.

## Scope

Define options, required inputs, path resolution, JSON/human streams, stable IDs, additive versus
breaking schema changes, and signal behavior. `pass_with_warnings` exits 0. Do not implement dispatch.

## Sequence

1. Write command and global-option grammar.
2. Add pass/warning/fail/invalid/capability/internal fixtures.
3. Define stdout, stderr, report-file equivalence, and path representation.
4. Add fixture/schema validation tests.

## Acceptance And Verification

- Every stable command and privacy-sensitive option documented.
- Machine stdout fixture parses as one JSON value with no logs.
- Exit matrix covers every terminal category.
- Windows and POSIX path fixtures exist.
- `npm test -- --test-name-pattern="CLI|schema|exit"`

## Failure And Privacy Constraints

No absolute developer path, raw resume/JD content, or secret appears in fixtures. Avoid making warning
status a process failure.

## Handoff Evidence

Grammar, schema version, fixtures, exit matrix, compatibility rules, and downstream implementation
questions resolved.
