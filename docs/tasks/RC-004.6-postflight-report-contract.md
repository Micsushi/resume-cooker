# RC-004.6: Versioned Postflight Report Contract

## Outcome

All postflight checks aggregate into one sanitized schema-v1 report whose status and exit behavior
match D7.

## Status Gate

**Blocked by RC-004.2 through RC-004.5 and RC-006.1.**

## Owned Surfaces

- `checker/scripts/compare.mjs`
- `checker/scripts/compare-lib.mjs`
- `checker/scripts/report-lib.mjs`
- report and compare tests
- `docs/evaluation-suites.md`

## Inputs And Outputs

Input: normalized fact, retention, grounding, and artifact findings. Output: schema-v1 comparison
report with stable IDs, categories, severity, status, sanitized evidence, confidence, inputs checked,
skips, and content-left-machine metadata.

## Scope

Distinguish source unavailable, tailored artifact unavailable, confirmed regression, review warning,
and optional skip. Map `pass`/`pass_with_warnings` to exit 0 and `fail` to exit 2. Reserve invalid,
capability, and internal exits per D7. Do not package CLI dispatch here.

## Sequence

1. Freeze passing, warning, failure, invalid, and capability report fixtures.
2. Add schema version and required fields.
3. Aggregate check families without hiding incomplete evidence.
4. Validate evidence sanitizer and exit mapping.
5. Document compatibility rules.

## Acceptance And Verification

- Report and process exit agree for every fixture.
- Missing/failed checks cannot produce implicit pass.
- No absolute host path or raw private value survives sanitizer tests.
- `node --test checker/scripts/compare-lib.test.mjs checker/scripts/report-lib.test.mjs`
- Run valid and regression compare commands and parse JSON.

## Failure And Privacy Constraints

Never change stable IDs silently, mix source defects with tailoring regressions, or expose raw input.

## Handoff Evidence

Schema fixture set, exit matrix, compatibility notes, sanitizer tests, and downstream RC-007/RC-008
requirements.
