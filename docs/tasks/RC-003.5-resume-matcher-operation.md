# RC-003.5: Resume-Matcher Bounded Operation

## Outcome

Resume-Matcher either provides one reproducible, useful local evidence operation or receives a
documented deferral with measured setup cost and revisit trigger.

## Status Gate

**Blocked by RC-003.1 and completion of RC-003.2 through RC-003.4.** Requires local snapshot
inspection before choosing execution surface.

## Owned Surfaces

- `checker/scripts/tester-wrapper.mjs`
- Resume-Matcher adapter tests
- `testers/Resume-Matcher/` documentation only unless a minimal compatibility fix is required
- `testers/README.md`

## Inputs And Outputs

Input: public fixture and candidate local operation. Output: executed normalized evidence or a
deferral record naming blockers, cost, and revisit condition.

## Scope

Inventory import requirements, database/network dependencies, startup cost, and smallest useful
backend function. Prefer a parser or deterministic comparison operation. Import smoke alone does not
count as evidence. Do not start persistent services or alter its product architecture.

## Sequence

1. Measure current import smoke and dependency surface.
2. Evaluate one bounded operation against adapter contract.
3. Implement/test adapter if setup stays bounded.
4. Otherwise document deferral and keep explicit skip.

## Acceptance And Verification

- Outcome is executed evidence or explicit accepted deferral, never ambiguous “supported.”
- Any process terminates and cleans state.
- `npm test -- --test-name-pattern="Resume-Matcher"`
- Exact candidate command or deferral evidence recorded.

## Failure And Privacy Constraints

No hosted service, database migration, persistent state, private artifact, or unbounded install is
introduced.

## Handoff Evidence

Measured dependency/startup cost, selected operation and result or deferral rationale, cleanup, and
revisit trigger.
