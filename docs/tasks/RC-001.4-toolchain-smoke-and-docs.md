# RC-001.4: Toolchain Smoke Evidence And Documentation

## Outcome

Real unavailable and usable toolchain paths match the structured capability contract on at least one
supported host.

## Status Gate

**Blocked by RC-001.2 and RC-001.3.** Usable-path evidence additionally requires native TeX/Poppler
or a running Docker daemon.

## Owned Surfaces

- `generator/README.md`
- `MACOS_COMPATIBILITY.md`
- `LINUX_COMPATIBILITY.md`
- private project handoff

## Inputs And Outputs

Input: host tool versions and daemon state. Output: sanitized command matrix and reproducible setup or
recovery instructions.

## Scope

Run probes with Docker stopped and running when available, plus any native PDF tools. Exercise
`--require-pdf-engine`, explicit Docker build, extraction, and page counting. Document observed
behavior without committing generated artifacts or host paths.

## Sequence

1. Capture OS, architecture, Node/npm, and tool versions.
2. Run unavailable-capability matrix.
3. Run usable build/extraction matrix when a runtime exists.
4. Update compatibility troubleshooting.
5. Record skipped paths and exact reasons privately.

## Acceptance And Verification

- Stopped Docker produces `available: true`, `usable: false` and nonzero strict probe.
- Running path, if available, produces non-empty output verified by the caller.
- Docs name recovery steps for terminal and GUI PATH differences.
- `npm run check:tools`
- `npm run check:tools -- --require-pdf-engine`
- `git status --short`

## Failure And Privacy Constraints

Do not paste usernames, socket paths, resume data, PDFs, logs, or credentials into public evidence.

## Handoff Evidence

Version table, stopped/running results, skipped paths, ignored artifact list, and next reproduction
command.
