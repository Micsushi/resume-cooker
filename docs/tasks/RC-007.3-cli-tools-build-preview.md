# RC-007.3: CLI Tools, Build, And Preview Commands

## Outcome

Stable CLI commands expose tool inventory, intentional PDF build, and loopback preview through
existing core modules.

## Status Gate

**Blocked by RC-001.3, RC-002.3, and RC-007.2.**

## Owned Surfaces

- CLI command handlers
- generator module exports needed by handlers
- command-level tests
- `package.json` compatibility aliases

## Inputs And Outputs

Input: validated command request. Output: structured capability/build/preview result, intentional
artifact, or categorized capability/internal error.

## Scope

Thinly dispatch to shared tooling; do not duplicate engine or preview logic. Build verifies non-empty
output. Preview binds loopback, reports port/source safely, and cleans children on interruption.

## Sequence

1. Add injected command fixtures for pass/capability/internal outcomes.
2. Implement `tools` JSON/human handler.
3. Implement `build` handler and artifact verification.
4. Implement `preview` lifecycle handler.
5. Point npm aliases through stable boundary where safe.

## Acceptance And Verification

- `npx resume-cooker tools --json` emits parseable JSON only.
- Missing engine exits 69; bad source exits 64.
- Build success verifies output; preview interruption leaves no process.
- Relevant CLI tests, `npm run check:tools`, and `npm run lint`.

## Failure And Privacy Constraints

Do not make optional runtimes package dependencies, bind LAN by default, or mix human logs into JSON.

## Handoff Evidence

Command matrix, capability error, build artifact proof, preview cleanup, and npm alias compatibility.
