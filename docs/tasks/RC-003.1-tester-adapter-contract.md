# RC-003.1: Common Tester Adapter Contract

## Outcome

Every vendored tester runs through one normalized adapter boundary with explicit execution, skip,
timeout, failure, and sanitization semantics.

## Status Gate

**Ready.** Uses injected runners and synthetic outputs; no tester dependencies required.

## Owned Surfaces

- `checker/scripts/tester-wrapper.mjs`
- `checker/scripts/tester-runner.mjs`
- tester wrapper/runner tests
- `testers/manifest.json`
- `testers/README.md`

## Inputs And Outputs

Input: adapter ID, public artifact paths, runtime capability, timeout, injected runner. Output:
normalized finding IDs, execution state, sanitized metadata, duration, and skip/failure reason.

## Scope

Define required inputs, command construction, timeout/cancellation, output validation, cleanup, and
states `executed_pass`, `executed_warning`, `executed_fail`, and `skipped`. Map skips to explicit
report findings without treating them as passes. Do not integrate a specific tester here.

## Sequence

1. Freeze adapter result schema and manifest fields.
2. Add fixtures for missing input/runtime, timeout, nonzero exit, malformed output, pass, and warning.
3. Implement shared runner and sanitization.
4. Make report aggregation preserve execution state.

## Acceptance And Verification

- Every skip has stable ID and exact reason.
- Requested/required adapter failure cannot silently downgrade to pass.
- Timeout cleans child process.
- `node --test checker/scripts/*tester*.test.mjs`
- `npm run check:testers` without dependencies; expect explicit skips.

## Failure And Privacy Constraints

No raw resume/JD text, contact values, full child stderr, virtualenv path, or API key enters reports.

## Handoff Evidence

Schema example, fixture matrix, timeout value, sanitizer rules, and compatibility note for all four
snapshots.
