# RC-005.1: Live API Smoke Guardrails

## Outcome

Live provider tests use verified synthetic fixtures, explicit authorization, bounded requests/cost,
safe secret injection, and a repeatable evidence checklist.

## Status Gate

**Blocked until the user authorizes live requests, provider scope, and spend/request ceilings.** No
key or environment variable implies authorization.

## Owned Surfaces

- public synthetic resume/JD fixtures
- `docs/evaluation-suites.md`
- `docs/tasks/RC-005-api-validation.md`
- private project handoff

## Inputs And Outputs

Input: approved providers, scoped credentials, request ceiling, spend ceiling, timeout/input/output
limits, and reviewed synthetic fixtures. Output: signed-off smoke matrix and safe command template.

## Scope

Audit fixtures for real identities and unique claims, define secret-loading and cleanup, disable shell
tracing, cap requests, and define evidence that excludes keys/prompts/responses. Do not make a live
request in this package.

## Sequence

1. Review fixtures and record synthetic-data confirmation.
2. Obtain explicit provider and budget authorization.
3. Define environment limits and per-case request count.
4. Define console/report secret audit and cleanup.
5. Freeze provider smoke matrix.

## Acceptance And Verification

- Authorization names provider, maximum requests, and maximum spend.
- Fixture audit finds no real personal values.
- Disabled/key-present and oversized-input cases require zero requests.
- `npm test -- --test-name-pattern="API|provider"`
- Secret-pattern scan command documented without literal credentials.

## Failure And Privacy Constraints

Never paste keys into docs/history, send private defaults, enable retries beyond budget, or retain raw
provider content by default.

## Handoff Evidence

Authorization scope, budgets, fixture confirmation, safe environment template, cleanup, and exact
request accounting plan.
