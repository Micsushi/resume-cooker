# RC-008.4: Hunt Postflight Invocation

## Outcome

Hunt runs Resume Cooker comparison after Fletcher produces final artifacts and before C3 readiness is
decided.

## Status Gate

**Blocked by RC-004.6, RC-008.2, and a stable Fletcher output contract from RC-008.1.**

## Owned Surfaces

- Hunt Fletcher coordinator/artifact/test paths from RC-008.1
- postflight synthetic fixtures
- integration documentation

## Inputs And Outputs

Input: original source/structured facts, tailored source/structured facts, final PDF, optional JD,
protected/omitted IDs, Fletcher flags. Output: validated postflight report reference and typed run
result.

## Scope

Invoke compare only after successful Fletcher artifacts exist. Preserve original and tailored
references, supply structured facts with D4 precedence, and prevent Fletcher failure from appearing
as postflight success. Do not decide C3 readiness here.

## Sequence

1. Add successful, Fletcher-failed, missing-artifact, regression, and timeout fixtures.
2. Build postflight request from actual artifact references.
3. Invoke adapter and retain report identity.
4. Ensure cancellation/error cleanup.
5. Record whether PDF checks and optional testers ran.

## Acceptance And Verification

- Postflight occurs after final artifact creation and before readiness.
- Missing/failing Fletcher output cannot yield postflight pass.
- Structured/source/PDF inputs match D4.
- Hunt integration tests for successful, regression, missing, timeout, and cancellation flows.

## Failure And Privacy Constraints

Do not duplicate artifacts unnecessarily or persist raw source/report content outside approved local
storage.

## Handoff Evidence

Call-order proof, input mapping, artifact references, fixture results, cleanup, and skipped optional
checks.
