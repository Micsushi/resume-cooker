# RC-007.6: CLI Package And Cross-Platform Verification

## Outcome

A fresh checkout/package can invoke the documented CLI on Windows, Linux, and macOS without shipping
private/generated artifacts or requiring optional runtimes.

## Status Gate

**Blocked by RC-007.5.** Requires smoke access to supported platforms; unit path coverage may precede
all hosts.

## Owned Surfaces

- `package.json`, package inclusion/exclusion files
- `README.md`, `docs/cli-contract.md`
- CI configuration for lightweight CLI tests
- private handoff for host smoke evidence

## Inputs And Outputs

Input: fresh install/package tarball and platform matrices. Output: verified file list, help/tools/check
smokes, compatibility statement, and untested-platform disclosure.

## Scope

Run `npm ci`, package dry run, invoke CLI without global install, inspect tarball, and exercise
network-free public fixtures. Do not publish npm package or install TeX/tester dependencies in root CI.

## Sequence

1. Add lightweight CLI tests to root CI.
2. Inspect `npm pack --dry-run` contents.
3. Test fresh-checkout help/tools/local fixture.
4. Record Windows/Linux/macOS results.
5. Document npm alias migration and compatibility policy.

## Acceptance And Verification

- Package excludes `.runtime`, outputs, keys, logs, agent config, and vendored dependency artifacts.
- Help/local checks work from clean install.
- `npm ci`, `npm pack --dry-run`, `npx resume-cooker --help`, `npm run ci`.

## Failure And Privacy Constraints

Do not publish, bundle native tools, or claim an untested platform verified.

## Handoff Evidence

Tarball file list, platform table, fresh-checkout commands, npm alias status, excluded artifact audit,
and remaining limitations.
